# STAR Story

## Situation

Payment companies need fraud systems that are fast, explainable, monitored, and reliable. A notebook model does not prove the platform can support production fraud operations.

## Task

Build a local end-to-end fraud feature store and MLOps pipeline that transforms synthetic transactions into trusted features, trains a model, scores fraud risk, creates alerts, and monitors drift.

## Action

Implemented synthetic data generation, fraud pattern injection, data quality checks, quarantine, point-in-time feature engineering, DuckDB feature store, model training, model registry, scoring, reason codes, alert queue, monitoring reports, FastAPI, Streamlit, tests, CI, and Docker.

## Result

Created a reproducible portfolio project showing enterprise-style fraud foundations: feature reliability, model metrics, risk scoring, reason codes, investigator workflows, and monitoring evidence.
